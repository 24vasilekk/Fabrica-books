# Техническое задание
## Проект: Telegram WebApp / сайт «Фабрика воспоминаний books»

## 1. Общая идея проекта
Нужно разработать Telegram WebApp (или адаптивный сайт, открываемый внутри Telegram) для заказа персонализированной книги.

Пользовательский путь:
1. Пользователь заходит на сайт/веб-приложение.
2. Выбирает, для кого нужна книга.
3. Оплачивает заказ онлайн или вводит код, если оплатил через менеджера.
4. Отвечает на 100 вопросов анкеты.
5. Загружает до 50 фотографий.
6. Нейросеть автоматически генерирует полный текст книги на основе ответов.
7. Готовый текст отправляется команде «Фабрика воспоминаний books».
8. Команда создает макет для печати и передает его в типографию.

## 2. Название проекта
`Фабрика воспоминаний books`

## 3. Формат продукта
Продукт должен работать как:
- Telegram WebApp;
- адаптивный сайт для мобильных устройств;
- корректно отображаться на десктопе.

Приоритет: мобильная версия внутри Telegram.

## 4. Визуальный стиль
Оформление должно быть выполнено в стилистике сайта «Фабрика воспоминаний».
Референс по стилю:
- светлый фон с мягкими градиентами;
- премиальный, аккуратный, эмоциональный визуальный язык;
- акцент на атмосферу тепла, памяти, личной истории;
- карточки и формы с мягкими скруглениями;
- легкие тени, полупрозрачные блоки, спокойная анимация;
- ощущение персонального дорогого подарка.

Дополнительно по интерфейсу:
- дизайн должен быть современным и минималистичным;
- основные цвета: молочный, светло-бежевый, пудровый, серо-графитовый;
- акцентный цвет можно использовать мягкий красный/бордовый в духе основного бренда;
- типографика: крупные эмоциональные заголовки + аккуратный читаемый текст;
- все формы должны быть простыми и понятными для пользователя.

Важно:
Новый продукт должен визуально ассоциироваться с брендом «Фабрика воспоминаний», но быть отдельным направлением именно для книг.

## 5. Цель сервиса
Автоматизировать сбор истории клиента для создания персонализированной книги и упростить передачу материалов в работу.

## 6. Основные разделы / экраны

### 6.1. Главный экран
На первом экране пользователь видит:
- название: `Фабрика воспоминаний books`;
- короткий эмоциональный оффер;
- описание продукта;
- кнопку начала заказа.

Пример смысла оффера:
- «Создайте книгу, в которой живут ваши чувства, воспоминания и важные моменты».

На первом экране также можно показать:
- для кого подходит книга;
- как это работает;
- почему это ценный подарок;
- кнопку `Начать` / `Создать книгу`.

### 6.2. Экран выбора типа книги
Пользователь выбирает, для кого создается книга.

Варианты:
- для любимого человека;
- для мамы;
- для папы;
- для ребенка;
- для друга / подруги;
- семейная книга;
- свой вариант.

Для каждого варианта можно показать:
- название;
- короткое описание;
- атмосферную карточку.

После выбора пользователь нажимает `Продолжить`.

### 6.3. Экран оплаты
Пользователь должен выбрать один из сценариев:
- оплатить онлайн;
- ввести код, если уже оплатил через менеджера.

#### Вариант 1. Онлайн-оплата
Нужно подключить оплату через платежную систему.
После успешной оплаты пользователь автоматически переходит к анкете.

#### Вариант 2. Оплата через менеджера
Если пользователь уже оплатил заказ через менеджера, он вводит специальный код доступа.

Логика:
- поле для ввода кода;
- кнопка `Проверить код`;
- если код валиден, открыть доступ к анкете;
- если код невалиден, показать понятную ошибку.

### 6.4. Экран анкеты (100 вопросов)
После подтверждения оплаты пользователь переходит к большой анкете.

Требования к анкете:
- всего 100 вопросов;
- пошаговое прохождение;
- индикатор прогресса;
- возможность вернуться к предыдущему вопросу;
- возможность пропустить вопрос, если это допустимо;
- автосохранение ответов;
- сохранение прогресса, если пользователь закрыл приложение;
- поддержка разных типов вопросов.

#### Типы вопросов:
- короткий текст;
- длинный текст;
- выбор одного варианта;
- выбор нескольких вариантов;
- дата;
- числовое значение;
- загрузка дополнительного комментария.

#### Дополнительные требования:
- вопросы можно разбить по главам/блокам;
- сверху показывать: номер вопроса, номер главы, прогресс;
- тексты должны быть эмоциональными и простыми;
- в конце нужен экран предпросмотра ответов.

Пример логики отображения:
- `Глава 2/5`;
- `Вопрос 18/100`;
- прогресс-бар;
- поле ответа;
- кнопки `Назад`, `Пропустить`, `Далее`.

### 6.5. Экран обзора ответов
После прохождения анкеты пользователь видит сводку:
- список вопросов;
- заполненные/пропущенные ответы;
- возможность вернуться и изменить любой ответ.

После подтверждения пользователь нажимает:
- `Продолжить к загрузке фотографий`.

### 6.6. Экран загрузки фотографий
Пользователь загружает фотографии для книги.

Требования:
- максимум 50 фото;
- отображение количества загруженных файлов;
- превью изображений;
- возможность удалить фото;
- возможность изменить порядок фото;
- допустимые форматы: JPG, JPEG, PNG, HEIC при возможности;
- ограничение по размеру файлов;
- подсказка по качеству фото.

Дополнительно:
- можно добавить поле комментария к фото или блоку фотографий;
- можно добавить рекомендации: «загружайте значимые фото в хорошем качестве».

После загрузки пользователь нажимает:
- `Завершить и отправить материалы`.

### 6.7. Экран финального подтверждения
Пользователь видит сообщение:
- что анкета и фото успешно отправлены;
- что текст книги будет автоматически сформирован;
- что далее команда подготовит макет и передаст его в печать.

Пример блоков:
- статус заказа;
- номер заказа;
- ориентировочный срок обработки;
- кнопка связи с менеджером.

## 7. Бизнес-логика

### 7.1. Сценарий доступа
Доступ к анкете и загрузке фото открывается только после:
- успешной онлайн-оплаты; или
- успешной проверки кода от менеджера.

### 7.2. Логика сохранения
Нужно сохранять:
- выбранный тип книги;
- статус оплаты;
- введенный код;
- все ответы пользователя;
- прогресс анкеты;
- список загруженных фотографий;
- дату и время отправки.

### 7.3. Генерация текста нейросетью
После отправки анкеты и фото система должна:
1. собрать все ответы пользователя;
2. сформировать структурированный prompt для нейросети;
3. сгенерировать полный текст книги;
4. сохранить результат в админке/базе;
5. отправить результат команде проекта.

Требования к генерации:
- текст должен быть литературным, теплым, эмоциональным;
- учитывать, для кого создается книга;
- использовать ответы пользователя как основу сюжета;
- не придумывать критически важные факты, если их нет в анкете;
- при нехватке данных заполнять текст аккуратно и нейтрально;
- структура текста должна быть пригодна для дальнейшей верстки книги.

Желательно, чтобы AI возвращал:
- название книги;
- вступление;
- структуру по главам;
- основной текст;
- финальное послание.

### 7.4. Передача результата команде
После генерации система должна отправить менеджеру / команде:
- ID заказа;
- имя клиента;
- контакт клиента;
- тип книги;
- статус оплаты;
- все ответы анкеты;
- ссылки на фото;
- автоматически сгенерированный текст;
- дату создания заказа.

Возможные способы передачи:
- Telegram-бот в служебный чат;
- email;
- CRM / таблица / админ-панель.

Приоритетный вариант:
- отправка в Telegram-бот / внутренний чат команды.

## 8. Роли пользователей

### 8.1. Клиент
Может:
- выбрать тип книги;
- оплатить или ввести код;
- пройти анкету;
- загрузить фото;
- отправить материалы.

### 8.2. Администратор / менеджер
Может:
- создавать и выдавать коды доступа;
- видеть список заказов;
- открывать ответы клиента;
- получать сгенерированный текст;
- менять статусы заказа;
- передавать заказ в макет / печать.

## 9. Админ-функционал
Нужен минимальный административный функционал.

### Обязательные возможности:
- авторизация администратора;
- список заказов;
- карточка заказа;
- статусы заказа.

### Статусы заказа:
- новый;
- оплачен;
- анкета в процессе;
- анкета завершена;
- фото загружены;
- текст сгенерирован;
- макет в работе;
- отправлено в печать;
- завершен.

### В карточке заказа отображать:
- номер заказа;
- дата;
- имя клиента;
- контакт;
- тип книги;
- способ оплаты;
- код менеджера (если был);
- ответы на анкету;
- загруженные фото;
- сгенерированный текст;
- комментарий менеджера.

## 10. Технические требования

### 10.1. Frontend
Нужно реализовать:
- адаптивный интерфейс;
- корректную работу в Telegram WebApp;
- быстрые переходы между шагами;
- плавный UX;
- валидацию форм.

### 10.2. Backend
Нужно реализовать:
- хранение заказов;
- хранение ответов;
- загрузку и хранение фото;
- проверку кодов доступа;
- интеграцию с оплатой;
- интеграцию с AI для генерации текста;
- отправку данных менеджеру.

### 10.3. База данных
Нужно хранить:
- пользователей;
- заказы;
- статусы заказов;
- ответы на вопросы;
- фото;
- коды доступа;
- сгенерированные тексты.

### 10.4. Интеграции
Предусмотреть интеграции:
- Telegram Bot API / Telegram WebApp;
- платежная система;
- AI API для генерации текста;
- облачное хранилище файлов;
- внутренний Telegram-чат или CRM.

## 11. Нефункциональные требования
- сайт должен быстро загружаться;
- интерфейс должен быть интуитивным;
- все данные должны сохраняться без потерь;
- пользователь не должен терять прогресс;
- фото должны загружаться стабильно;
- система должна быть готова к расширению;
- тексты ошибок должны быть понятными;
- дизайн должен вызывать доверие и ощущение премиального продукта.

## 12. Контентные блоки на главной странице
На главной можно предусмотреть следующие блоки:
- Hero-блок с оффером и кнопкой старта;
- Для кого эта книга;
- Как это работает;
- Что входит в результат;
- Примеры сценариев книг;
- FAQ;
- Блок связи с менеджером;
- Кнопка перехода в Telegram.

## 13. Пример пользовательского сценария
1. Пользователь открывает `Фабрика воспоминаний books`.
2. Нажимает `Создать книгу`.
3. Выбирает вариант `Для любимого человека`.
4. Оплачивает заказ или вводит код от менеджера.
5. Заполняет 100 вопросов анкеты.
6. Проверяет свои ответы на экране обзора.
7. Загружает до 50 фотографий.
8. Подтверждает отправку.
9. Система генерирует текст книги.
10. Команда получает материалы, делает макет и передает в типографию.

## 14. Что должно быть на выходе у разработчика

### MVP версия должна включать:
- главный экран;
- выбор типа книги;
- оплату или ввод кода;
- анкету на 100 вопросов;
- обзор ответов;
- загрузку до 50 фото;
- отправку данных;
- AI-генерацию текста;
- передачу результата менеджеру;
- базовую админку.

## 15. Отдельные пожелания по UX
- не перегружать пользователя;
- делать длинную анкету психологически легкой за счет глав, прогресса и аккуратного текста;
- показать, что пользователь создает не просто анкету, а будущую книгу;
- в ключевых местах добавлять эмоциональные подсказки;
- после каждого крупного шага показывать понятный следующий шаг.

## 16. Рекомендации по структуре вопросов
100 вопросов лучше разделить на 5 тематических блоков по 20 вопросов:
- знакомство и контекст;
- важные воспоминания;
- чувства и личные смыслы;
- детали, символы, атмосфера;
- послание и финал книги.

## 17. Что желательно предусмотреть сразу
- возможность редактировать список вопросов без участия разработчика;
- возможность менять тексты на лендинге;
- возможность добавлять новые типы книг;
- возможность повторной генерации текста;
- возможность ручной правки сгенерированного текста в админке.

## 18. Итог
Нужно создать красивый, эмоциональный и удобный Telegram WebApp / сайт в стиле бренда «Фабрика воспоминаний» для заказа персонализированных книг. Ключевая задача сервиса: провести клиента по понятному пути от выбора книги и оплаты до заполнения анкеты, загрузки фото и автоматической генерации текста книги для дальнейшей подготовки макета и печати.
